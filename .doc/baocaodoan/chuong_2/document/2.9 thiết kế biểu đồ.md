# 2.9. Hệ thống biểu đồ thiết kế

Mục này tổng hợp các biểu đồ quan trọng phục vụ việc mô tả hệ thống theo góc nhìn phân tích và thiết kế.

## 2.9.1. Biểu đồ thành phần

```mermaid
flowchart LR
    FE[Frontend Next.js]
    AUTH[Auth API]
    TASK[Task API]
    NOTE[Note API]
    FIN[Finance API]
    FOCUS[Focus API]
    CHAT[Chat API]
    ADMIN[Admin API]
    DB[(SQL Server)]
    R[Rasa]

    FE --> AUTH
    FE --> TASK
    FE --> NOTE
    FE --> FIN
    FE --> FOCUS
    FE --> CHAT
    FE --> ADMIN

    AUTH --> DB
    TASK --> DB
    NOTE --> DB
    FIN --> DB
    FOCUS --> DB
    ADMIN --> DB
    CHAT --> DB
    CHAT --> R
```

## 2.9.2. Biểu đồ trạng thái công việc

```mermaid
stateDiagram-v2
    [*] --> Todo
    Todo --> InProgress: Bắt đầu thực hiện
    InProgress --> Completed: Hoàn thành
    InProgress --> Todo: Hoãn lại
    Todo --> Completed: Hoàn thành trực tiếp
    Completed --> [*]
```

## 2.9.3. Biểu đồ tuần tự cho quản trị người dùng

```mermaid
sequenceDiagram
    actor A as Quản trị viên
    participant FE as Frontend
    participant API as AdminUsersController
    participant ID as Identity
    participant DB as Database

    A->>FE: Mở danh sách người dùng
    FE->>API: GET /api/AdminUsers
    API->>DB: Truy vấn người dùng và vai trò
    DB-->>API: Dữ liệu phân trang
    API-->>FE: Danh sách người dùng
    A->>FE: Khóa tài khoản
    FE->>API: POST /api/AdminUsers/{id}/ban
    API->>ID: Cập nhật lockout
    ID-->>API: Thành công
    API-->>FE: Trạng thái mới
```

## 2.9.4. Biểu đồ hoạt động quản lý chi tiêu

```mermaid
flowchart TD
    A[Bắt đầu] --> B[Người dùng nhập khoản chi]
    B --> C[Kiểm tra JWT]
    C --> D[Kiểm tra danh mục]
    D --> E{Danh mục tồn tại?}
    E -- Không --> F[Trả lỗi]
    E -- Có --> G[Kiểm tra số tiền]
    G --> H{Hợp lệ?}
    H -- Không --> F
    H -- Có --> I[Lưu FinanceEntry]
    I --> J[Cập nhật danh sách và thống kê]
    J --> K[Kết thúc]
```

## 2.9.5. Ý nghĩa sử dụng biểu đồ

- Biểu đồ ngữ cảnh và DFD giúp mô tả ranh giới hệ thống.
- Biểu đồ use case và hoạt động giúp mô tả hành vi nghiệp vụ.
- Biểu đồ tuần tự giúp làm rõ thứ tự tương tác giữa các thành phần.
- Biểu đồ trạng thái giúp mô tả vòng đời đối tượng chính như `TaskItem`.
- Biểu đồ ERD và thành phần giúp hỗ trợ thiết kế triển khai thực tế.
